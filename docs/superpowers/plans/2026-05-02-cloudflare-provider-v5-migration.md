# Cloudflare Provider 4.52.7 → 5.x Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the repo from Cloudflare TF provider `~> 4.40` (locked at `4.52.7`) to `~> 5.0`, with no DNS outage and a clean no-op `terraform plan`.

**Architecture:** Single PR on branch `feat/cloudflare-provider-v5`. Use Cloudflare's official `tf-migrate` v1.0.1 CLI to transform HCL (resource type renames, attribute updates, `moved {}` blocks). The v5 provider's built-in state upgraders handle in-state schema migration during `terraform plan`/`apply`. Pre-merge gate is a clean local plan; CI plan is the second gate.

**Tech Stack:** Terraform 1.10.5, Cloudflare TF provider 5.x, [tf-migrate v1.0.1](https://github.com/cloudflare/tf-migrate), Docker (for local terraform execution since Terraform isn't installed natively).

**Spec:** `docs/superpowers/specs/2026-05-02-cloudflare-provider-v5-migration-design.md`

## Task 4: Run tf-migrate for real

**Files:** all 6 `cloudflare.tf` files, the module's `main.tf`, all 7 `versions.tf` files. Edits made by `tf-migrate`.

- [ ] **Step 1: Run the migration**

```bash
cd terraform

cd ..
```

Expected output: success message, list of files modified.

- [ ] **Step 2: Inspect the diff**

```bash
git diff --stat
git diff terraform/modules/cloudflare-default-server-subdomains/main.tf
```

Verify:

- Module's `cloudflare_record.a` and `cloudflare_record.aaaa` became `cloudflare_dns_record.a` and `cloudflare_dns_record.aaaa`.
- All `versions.tf` files now pin `~> 5.0`.
- `moved {}` blocks present (likely in caller files, since the module-internal rename moves through the existing `module.cf_proxied_*.cloudflare_record.<x>` addresses).

- [ ] **Step 3: Manual sanity edits if tf-migrate misses something**

Possible issues to look for and fix manually:

- **`name` attribute not FQDN-converted:** v5 requires the full FQDN. Caller code passes `each.value` (e.g. `"@"`, `"www"`, `"mail-beta"`) to the module. If tf-migrate doesn't auto-convert `name = each.value` to `name = "${each.value}.${zone_name}"`, we need to thread the zone name into the module.
  - If module needs a zone-name input, add `variable "zone_name" { type = string }` to `terraform/modules/cloudflare-default-server-subdomains/variables.tf` and have main.tf compute `name = each.value == "@" ? var.zone_name : "${each.value}.${var.zone_name}"`. Each caller would then pass `zone_name = data.cloudflare_zone.<x>.name`.
  - Apply the same FQDN handling to the inline records (DKIM CNAMEs, MX, TXT) in `rentivo/cloudflare.tf` and `jorgejunior/cloudflare.tf`.
- **`value` → `content` not auto-renamed for non-DNS-record resources:** unlikely; spec says tf-migrate handles all attribute renames for `cloudflare_dns_record`.
- **Removed attributes (`hostname`, `allow_overwrite`):** we don't use these; should be no-op.

If you make manual edits, document each one in the eventual PR description.

- [ ] **Step 4: Format and validate locally**

```bash
docker run --rm -v "$PWD":/work -w /work/terraform hashicorp/terraform:1.10.5 fmt -recursive
docker run --rm -v "$PWD":/work -w /work/terraform hashicorp/terraform:1.10.5 init -upgrade -backend=false 2>&1 | tail -10
docker run --rm -v "$PWD":/work -w /work/terraform hashicorp/terraform:1.10.5 validate
```

Expected:
- `fmt -recursive` exits 0 (and may rewrite files; if so, run `fmt -check -recursive` after to confirm clean).
- `init -upgrade -backend=false` downloads cloudflare provider 5.x and rewrites `.terraform.lock.hcl`.
- `validate` returns `Success! The configuration is valid.`

If validate fails, the error message tells you exactly which resource/attribute needs fixing. Common ones:
- `Reference to undeclared resource: cloudflare_record` — somewhere still references the old name. Grep `terraform/` for `cloudflare_record` and update.
- `Argument expected here: name` — an attribute was renamed; check the v5 docs for the new name.
- Iterate until validate passes.
